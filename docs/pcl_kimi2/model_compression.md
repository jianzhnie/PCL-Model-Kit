## 什么是 LLM Compressor？

**LLM Compressor** 是一个用于优化大语言模型以适配 vLLM 部署的库，提供多种压缩算法来减小模型体积、降低硬件需求并提升推理性能。

### 核心优势

| 挑战 | 解决方案 |
| --- | --- |
| GPU 和基础设施成本 | 减少 50-75% 内存需求，支持更少 GPU 部署 |
| 响应延迟 | 量化权重加载更快，降低数据搬运开销 |
| 请求吞吐量 | 利用低精度张量核心加速计算 |
| 能源消耗 | 更小的模型推理功耗更低 |

### 支持的算法

| 算法 | 描述 | 适用场景 |
| --- | --- | --- |
| **RTN**（Round-to-Nearest） | 快速基线量化 | 最简配置的快速压缩 |
| **GPTQ** | 带校准的加权量化 | 高精度 4/8 位权重量化 |
| **AWQ** | 激活感知权重量化 | 保持重要权重精度 |
| **SmoothQuant** | W8A8 的异常值平滑 | 改善激活量化效果 |
| **SparseGPT** | 剪枝 + 量化 | 2:4 稀疏模式 |
| **SpinQuant** | 基于旋转的变换 | 改善低比特精度 |
| **QuIP** | 非相干处理 | 高级量化预处理 |
| **FP8 KV Cache** | KV 缓存量化 | Hopper+ GPU 长上下文推理 |
| **AutoRound** | 符号梯度下降优化舍入与裁剪 | 广泛兼容 |

### 支持的量化格式

| 格式 | 量化目标 | 最低计算能力 | 适用场景 |
| --- | --- | --- | --- |
| **W4A16 / W8A16** | 权重 | 8.0（Ampere+） | 较旧硬件上的延迟优化 |
| **W8A8-INT8** | 权重 + 激活 | 7.5（Turing+） | 性能与兼容性平衡 |
| **W8A8-FP8** | 权重 + 激活 | 8.9（Hopper+） | 现代 GPU 高吞吐量 |
| **NVFP4 / MXFP4** | 权重 + 激活 | 10.0（Blackwell） | 最新硬件最大压缩 |
| **W4AFP8** | 权重 + 激活 | 8.9（Hopper+） | 低比特权重 + 动态 FP8 激活 |
| **W4AINT8** | 权重 + 激活 | 7.5（Turing+） | 低比特权重 + 动态 INT8 激活 |

## PTQ 压缩方案详解

训练后量化（Post-Training Quantization）将可量化权重（如线性层）的精度降低到更低位宽。

### W8A8-FP8

- **原理**：权重和激活值均量化为 8 位浮点（FP8），通过 8 位算术运算实现约 **2 倍**权重压缩
- **权重量化**：逐通道或逐张量
- **激活量化**：动态逐 token（推荐）或静态逐张量，在 vLLM 推理期间完成
- **校准数据**：通常不需要（静态逐张量激活量化除外）
- **适用场景**：通用性能与压缩，尤其适合服务器端和批量推理
- **硬件要求**：NVIDIA GPU（计算能力 >= 9.0，即 Hopper/Blackwell），最新 AMD GPU

### W8A8-INT8

- **原理**：权重和激活值均量化为 8 位整型（INT8），约 **2 倍**权重压缩
- **权重量化**：GPTQ 逐通道或逐张量
- **激活量化**：动态逐 token 或静态逐张量（支持非对称量化），在 vLLM 推理期间完成
- **校准数据**：需要
- **适用场景**：通用性能与压缩，高 QPS 场景和离线推理速度提升显著
- **硬件要求**：所有 NVIDIA GPU（推荐计算能力 < 8.9，即 Ampere/Turing/Volta/Pascal）、AMD GPU、TPU、CPU 等

### W4A16

- **原理**：仅权重量化为 4 位整型（INT4），激活值保持 FP16，约 **3.7 倍**权重压缩
- **权重量化**：支持对称和非对称量化
- **推荐算法**：GPTQ（降低量化损失），也可使用 AWQ
- **校准数据**：需要
- **适用场景**：内存受限的延迟敏感应用，低 QPS 场景下通过更高压缩获得明显加速
- **硬件要求**：所有 NVIDIA GPU、AMD GPU、TPU、CPU 等

### W8A16

- **原理**：权重量化为 8 位整型，激活值保持 16 位整型
- **优势**：模型体积小于 FP32，原生 8 位整型硬件上推理更快，功耗和内存带宽更低
- **校准数据**：需要
- **硬件要求**：所有 NVIDIA GPU、AMD GPU、TPU、CPU 等

### NVFP4

- **原理**：NVIDIA Blackwell 架构引入的 4 位浮点格式，采用两级微块缩放策略保持数值精度
- **缩放机制**：每个张量生成全局缩放因子，每 16 个元素生成局部量化缩放（分组大小不可更改）
- **量化范围**：权重和激活值均生成全局缩放和局部量化缩放
- **校准数据**：需要
- **硬件要求**：NVIDIA Blackwell GPU 或更高版本

### W8A8-FP8_BLOCK

- **原理**：使用块级量化将权重压缩为 FP8（通常 128×128 分块），激活值使用动态逐 token 组（128）量化
- **激活量化**：在 vLLM 推理期间完成
- **校准数据**：需要（用于权重块级量化）
- **硬件要求**：所有 NVIDIA GPU、AMD GPU、TPU、CPU 等

## 稀疏化压缩

稀疏化通过将选定权重值剪枝为零来降低模型复杂度，同时保留关键权重。

### 2:4 半结构化稀疏

- **原理**：SparseGPT 算法，张量中每 4 个连续权重将 2 个置零
- **权重量化**：逐通道 8 位
- **激活量化**：动态逐 token 8 位
- **优势**：比 W8A8-FP8 推理效果更好，评估分数几乎无下降
- **注意**：小模型可能出现精度下降（非零权重不足以复现原始分布时）
- **校准数据**：需要
- **硬件要求**：所有 NVIDIA GPU 等（推荐计算能力 >= 9.0，即 Hopper/Blackwell）

### 非结构化稀疏

- **原理**：将个别权重置零，不强制固定模式，在权重贡献最小处移除
- **优势**：细粒度稀疏矩阵，优于块剪枝和通道剪枝
- **校准数据**：不需要
- **硬件要求**：所有 NVIDIA GPU、AMD GPU、TPU、CPU 等
